node概述

Posted on 2020-07-29 In 前端 , node

nodejs的中文社区

是什么

  1. nodejs是 js runtime，用于解析执行js代码，以前只有浏览器可以解析执行js代码
  2. node.js不是一门语言，不是库，也不是框架
  3. node是用来打开服务器黑盒的，除了node，其他的如java,PHP,Pyhton,Ruby,.Net都可以做到
    具体的API操作可以直接查看官方文档
    更多底层细节可参考：

    《深入浅出nodejs》

    特性

  4. event-driven
  5. non-blocking I/O model,即异步
  6. lightweight and efficient

    能做什么？

  7. 开发web服务器后台
  8. 命令行工具（webpack,gulp,npm)

    安装及常用命令

    查看版本号： 执行文件：文件不能用node.js命名
  • 浏览器中的JS没有文件操作能力；但是node中的js有,需要引入fs核心模块
  • node不太关注样式，更关注服务端的操作 第二个参数是回调函数，如果成功data有数据，error为null;如果失败，data为undifined, error有数据
  • npm是用nodejs开发的包管理工具，绝大多数的JS相关的包都存放在了npm上
  • 浏览器中的JS == ES + BOM + DOM
  • Node.js中的JS：只有ES，没有BOM，没有DOM
  • Node主要就是使用ES，核心模块进行编程

nginx

Posted on 2020-07-28 In 后端

是什么？

高性能的HTTP和反向代理服务器；特点是占有内存少，并发能力强

用途

  1. 反向代理：暴露的是代理服务器的IP，隐藏真实服务器的IP
  2. 负载均衡：把请求分发到各个服务器上;将负载分摊到不同服务器上，保证服务的可靠性，及响应的及时性
    nginx提供的分配策略
  • 轮询（默认）
    将请求按时间顺序逐一分配给不同的后端服务器，如果后端服务器down掉，可以自动剔除
  • weight
    weight越大，被分配的客户端越多
  • ip_hash
    每个请求按访问ip的hash结果分配，这样每个访客固定访问一个后端，可以解决session的问题，如：
  • fair(第三方）
    按后端服务器的响应时间来分配请求，响应时间短的优先分配
  1. 动静分离：动态请求和静态请求由不同的服务器解析，加快解析速度
    目前实现有两种：一是静态文件独立成单独的域名，放在独立的服务器上；二是动态和警惕啊文件混合在一起发布，通过nginx分开

主配置文件：nginx.conf
第一部分：全局块
worker_processes 1; # 生成的worker process数量
第二部分： events块
主要影响nginx服务器与用户的网络连接
events {
worker_connections 1024; # 表示每个work process支持的最大连接数为1024
}
第三部分：http块
代理、缓存、日志定义等功能和第三方模块配置均在这里
这里分为http全局块、server块

  • http全局块
    包括文件引入、MIME-TYPE定义、日志自定义、连接超时时间、单链接请求数上限

  • server块

    • 全局server块
      本虚拟机主机的监听配置&&本虚拟主机的名称or IP配置
    • location块
      一个server块可以配置多个location

其他零散

  1. location中expire参数可以设置有效时间，无修改从本地返回（304），有修改从服务器返回（200）
  2. master-workers机制的好处
    • 独立进程，不需要加锁，省掉了锁的开销
    • 一个进程退出不影响其他进程
    • 一个进程异常退出，只影响该进程负责的请求
